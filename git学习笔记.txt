1.git revert和git reset的区别

git revert （回滚）
会回滚指定的提交并产生一条新的提交记录，不会影响被回滚提交之后的提交内容，因此可能会产生冲突，冲突的原因是被回滚之前的内容和被回滚之后提交的内容有重叠，这很正常，手动解决就行。

git reset （重置）
会重置到指定的提交并将被回滚提交之后的提交内容清除，根据参数同，清除掉效果也不一样。参数有三个，[--soft | --mixed | --hard]
--mixed 为默认，可以不用带该参数，用于重置暂存区的文件与被指定的提交(commit)保持一致，工作区文件内容保持不变。
--soft 参数用于回退到某个版本，工作区文件内容和暂存区内容都保持不变
--hard 参数撤销工作区中所有未提交的修改内容，将暂存区与工作区都回到指定的版本，并删除被指定提交之后的所有信息提交。

git revert会撤销指定的提交
