Spring Boot

是一个简化Spring应用程序开发的框架，它可以让开发者快速构建和部署生产级的Spring应用

自动配置：Spring Boot基于一系列的条件判断自动配置Spring应用。例如，如果classpath下有spring-webmvc包，它会自动配置Spring MVC。

起步依赖 Starters：Spring Boot通过起步依赖来简化依赖管理。起步依赖是一组相关依赖的集合，它们被打包到一起，以便于开发者快速添加到项目中。例如，spring-boot-starter-web包含了构建Web应用所需的依赖。

无代码生成和XML配置 ：        Spring Boot不需要任何XML配置，也不需要额外的代码生成。大多数Spring Boot应用只需要很少的Spring配置。

内嵌服务器：Spring Boot提供了内嵌的Tomcat、Jetty或Undertow服务器，这使得开发者可以无需部署WAR文件到外部服务器即可运行应用。

命令行界面(LI）：Spring Boot CLI（命令行界面）是一个基于Groovy的命令行工具，可以用来快速构建Spring Boot应用原型。

Actuator:Spring Boot Actuator提供了生产环境中监控和管理应用程序的功能。它可以提供HTTP、JMX和SSH端点来查看应用程序的运行状态。

Spring Initializr:  这是一个在线工具，用于生成Spring Boot项目的初始结构。你可以选择项目的元数据（如项目名、描述、包名等），并选择所需的依赖，然后生成项目。

maven配置文件pom.xml，Spring Boot同样使用pom.xml进行配置

<dependencyManagement>
统一版本控制，该配置不会引入实际依赖，但会在子模块或当前模块中声明同 GAV（groupId、artifactId、version） 的 dependency 时强制使用指定版本，也就是说我们如果需要某个依赖，直接在我们的dependency中加入依赖就行，而无需指定版本。
 <type>pom</type>
 <scope>import</scope>
import作用域只能在<dependencyManagement>部分的<dependency>中使用
它只能用于type为pom的依赖
被导入的POM中的依赖管理配置会被合并到当前POM中

<parent>
引入标准父POM，从根本上改变了传统Maven项目中版本碎片化、配置冗余严重的局面。当一个项目声明 <parent> 引用该POM时，即自动继承了一整套经过严格验证的依赖版本组合、插件配置和构建属性，极大提升了开发效率与项目稳定性。

<dependencies>
项目中实际用到的依赖，如果已经在dependencyManagement（包括父pom中的dependencyManagement）定义了版本号，就无需指定版本号，从而使用集中定义的版本

二.springboot框架各层的关系和流程

1.仅仅有一个空的框架是没有用的，我们要给里面加些内容，这些内容一般分为四层，分别是controller层（控制层），service层（业务层），mapper层（数据持久层），model层（数据库实体层）。

2.controller层：它负责前后端交互，接受前端发送的请求（通过后端的路由地址发送），并调用service层，service层在返回数据给它，它在返回给前端。

service层：负责各种业务逻辑的处理，调用mapper层操作数据库，再对返回的数据进行各种业务上的处理，再返回给controll层。

mapper层：主要操作数据库，完成增删查改功能，把数据返回给service层。

model层：数据库的实体层，存放实体类，实现getset方法，是数据库的映射，变量的类型和名称必须和数据库相同，若名称由两个单词以上组成，第二个单词的首字母要大写。

3.springboot的流程

前端发送请求，controller层接受信息，然后调用service层的接口及其接口实现类，实现类在通过mapper层操作数据库，mapper层把数据返回给service层，然后在service层进行业务处理，然后在把数据返回给controller层。
